🎯 En bref

TypeScript est un sur-ensemble de JavaScript développé par Microsoft qui ajoute le typage statique optionnel. Le code TypeScript se compile en JavaScript standard. Il permet de détecter les erreurs à la compilation plutôt qu'à l'exécution, et améliore drastiquement l'expérience de développement (autocomplétion, refactoring, documentation).

Analogie

JavaScript c'est comme conduire sans tableau de bord : ça fonctionne, mais vous ne savez les problèmes qu'une fois sur la route. TypeScript c'est comme avoir un tableau de bord complet avec alertes : vous voyez les problèmes avant même de démarrer.

Utilisé par

Microsoft (VSCode, Azure), Google (Angular), Airbnb, Slack, Shopify, Discord — et la quasi-totalité des projets JavaScript modernes de taille significative.

JavaScript vs TypeScript

// JavaScript — aucune erreur détectée avant exécution
function add(a, b) {
    return a + b;
}
add("5", 3); // → "53" au lieu de 8, bug silencieux !

// TypeScript — erreur détectée à la compilation ✅
function add(a: number, b: number): number {
    return a + b;
}
add("5", 3); // ❌ Erreur : Argument of type 'string' is not assignable to type 'number'

Exemples pratiques

// Interface + types
interface User {
    id: number;
    name: string;
    email?: string;       // Optionnel
    role: 'admin' | 'user'; // Union type littéral
}

// Générique
function getFirst<T>(arr: T[]): T | undefined {
    return arr[0];
}
const first = getFirst([1, 2, 3]); // TypeScript infère : number

// Type guard
function formatId(id: string | number): string {
    if (typeof id === 'number') {
        return `#${id.toString().padStart(6, '0')}`; // TypeScript sait que id est number ici
    }
    return id; // TypeScript sait que id est string ici
}

// Async/await typé
async function fetchUser(id: number): Promise<User> {
    const response = await fetch(`/api/users/${id}`);
    return response.json() as User;
}

Configuration (tsconfig.json)

{
  "compilerOptions": {
    "target": "ES2020",
    "strict": true,          // Active tous les checks stricts (recommandé !)
    "noImplicitAny": true,   // Interdit les types 'any' implicites
    "strictNullChecks": true, // null et undefined doivent être gérés explicitement
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
