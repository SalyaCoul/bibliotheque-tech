🎯 En bref : Vite (prononcé "vit", comme le mot français) est un outil de build ultra-rapide pour les projets JavaScript modernes. Il a révolutionné le développement front-end en rendant le démarrage d'un projet quasi instantané.

Analogie : Avant Vite, démarrer un projet JavaScript c'était comme allumer une vieille voiture diesel par temps froid — on attendait que tout chauffe. Avec Vite, c'est comme appuyer sur le bouton d'une Tesla : instantané. La magie vient du fait que Vite ne "compile" plus tout le code au démarrage — il ne traite que ce dont vous avez besoin, à la demande.

Utilisé par : Vue.js (Vite est l'outil officiel de Vue), mais aussi React, Svelte, et n'importe quel projet JavaScript moderne. Créé par Evan You, le créateur de Vue.js.

Vite

Vite est un build tool (outil de construction) et un serveur de développement pour les applications JavaScript/TypeScript modernes.

Pourquoi Vite est-il si rapide ?

1. Pas de bundle au démarrage — Contrairement à Webpack qui doit "bundler" (empaqueter) tout votre code avant de démarrer, Vite exploite les ES Modules natifs du navigateur. Il sert directement les fichiers source, et le navigateur charge ce dont il a besoin.

2. Hot Module Replacement (HMR) ultrarapide — Quand vous modifiez un fichier, seul ce fichier est mis à jour dans le navigateur, pas toute l'application. La mise à jour est quasi instantanée, même sur des projets de grande taille.

3. Build optimisé avec Rollup — En production, Vite utilise Rollup pour créer des bundles optimisés et compressés.

Comparaison de démarrage

OutilTemps de démarrage (gros projet)
Webpack30-60 secondes
Vite< 1 seconde

Utilisation

# Créer un projet React avec Vite
npm create vite@latest mon-projet -- --template react

# Démarrer le serveur de développement
npm run dev

# Construire pour la production
npm run build

Frameworks qui utilisent Vite par défaut

  • Vue.js (outil officiel)
  • Nuxt.js
  • SvelteKit
  • Astro
  • React (alternative moderne à Create React App)
💡 Vite est devenu le standard de facto pour les nouveaux projets front-end JavaScript. Si vous démarrez un nouveau projet aujourd'hui, Vite est très probablement le bon choix.
