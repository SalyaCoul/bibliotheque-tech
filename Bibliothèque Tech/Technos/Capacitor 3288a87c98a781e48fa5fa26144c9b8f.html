🎯 En bref : Capacitor permet de transformer une application web (HTML/CSS/JavaScript) en une vraie application mobile iOS et Android, tout en accédant aux fonctionnalités natives du téléphone (caméra, GPS, notifications...).

Analogie : Imaginez que vous avez construit une belle maison (votre app web). Capacitor est comme un camion qui charge votre maison et la transporte dans l'App Store ou le Play Store, en la connectant aux tuyaux et câbles locaux (fonctionnalités du téléphone). Votre maison reste la même, mais elle fonctionne maintenant comme une vraie maison locale.

Utilisé par : Équipes qui ont une app web existante et veulent la publier sur mobile, développeurs Ionic, équipes Angular ou Vue qui veulent éviter d'apprendre React Native.

Capacitor

Capacitor est un runtime natif multiplateforme créé par l'équipe Ionic. Il permet de déployer des applications web sur iOS, Android et le web à partir d'un seul codebase.

Comment ça fonctionne

Capacitor embarque votre app web dans une WebView native (un navigateur intégré à l'app) et fournit des plugins JavaScript pour accéder aux APIs natives du téléphone :

Votre code web (HTML/CSS/JS)
         ↓
    Capacitor
   ↙         ↘
iOS (Swift)  Android (Kotlin/Java)

Plugins natifs disponibles

import { Camera } from '@capacitor/camera';
import { Geolocation } from '@capacitor/geolocation';
import { PushNotifications } from '@capacitor/push-notifications';

// Prendre une photo
const photo = await Camera.getPhoto({
  quality: 90,
  allowEditing: false,
  resultType: CameraResultType.Uri
});

// Obtenir la position GPS
const position = await Geolocation.getCurrentPosition();
console.log(position.coords.latitude, position.coords.longitude);

L'écosystème Ionic

Capacitor est souvent utilisé avec Ionic Framework — une librairie de composants UI adaptés au mobile (boutons, listes, onglets, modales...) qui ressemblent aux composants natifs iOS/Android.

# Créer un projet Ionic + Capacitor
npm install -g @ionic/cli
ionic start mon-app blank --type=react
ionic capacitor add ios
ionic capacitor add android

Points forts et limites

Points forts :

  • Réutilise une app web existante → gain de temps considérable
  • Fonctionne avec n'importe quel framework JS (React, Vue, Angular, Svelte...)
  • Plugins officiels bien maintenus
  • Accès aux plugins Cordova existants (compatibilité avec l'écosystème legacy)

Limites :

  • Les performances sont inférieures aux apps vraiment natives ou React Native
  • L'expérience utilisateur dépend de la qualité de la WebView (généralement très bonne sur les appareils récents)
  • Animations complexes peuvent être moins fluides
💡 Capacitor est le choix idéal si vous avez déjà une app web et voulez la publier sur mobile sans tout réécrire. Pour une app mobile conçue de zéro, React Native/Expo offre de meilleures performances et une expérience plus native.
