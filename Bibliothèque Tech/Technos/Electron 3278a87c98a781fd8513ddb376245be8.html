Electron

Statut
Terminé
TypeEnvironnement de Développement

🎯 En bref

Electron est un framework qui permet de créer des applications de bureau multiplateformes (Windows, macOS, Linux) avec des technologies web (HTML, CSS, JavaScript). Il combine Chromium (pour le rendu) et Node.js (pour l'accès système) dans un seul runtime.

Analogie

C'est comme construire une maison en utilisant des briques LEGO que vous connaissez déjà (HTML/CSS/JS) au lieu d'apprendre la maçonnerie native de chaque pays. Le résultat est une vraie maison qui fonctionne partout, même si elle pèse un peu plus lourd qu'une construction native.

Utilisé par

Visual Studio Code, Slack, Discord, Figma (anciennement), WhatsApp Desktop, Notion Desktop, Postman, 1Password.

Architecture Electron

┌─────────────────────────────────────────────────────┐
│                  Application Electron                │
├──────────────────────┬──────────────────────────────┤
│   Main Process       │      Renderer Process(es)    │
│   (Node.js)          │      (Chromium)               │
│                      │                              │
│ - Accès fichiers     │ - HTML / CSS / JavaScript    │
│ - Menu natif         │ - Framework (React, Vue...)  │
│ - Fenêtres (BrowserWindow) │ - UI de l'application  │
│ - Notifications      │                              │
│ - Mises à jour auto  │                              │
└──────────────────────┴──────────────────────────────┘
         ↕ Communication via IPC (Inter-Process Communication)

Exemple de base

// main.js — Processus principal (Node.js)
const { app, BrowserWindow, ipcMain } = require('electron');

app.whenReady().then(() => {
    const win = new BrowserWindow({
        width: 1200,
        height: 800,
        webPreferences: {
            preload: path.join(__dirname, 'preload.js') // Pont sécurisé
        }
    });

    win.loadFile('index.html'); // Ou loadURL pour une app web
});

// Écoute d'un message depuis le renderer
ipcMain.handle('read-file', async (event, filePath) => {
    return fs.readFileSync(filePath, 'utf-8');
});
// renderer.js — Interface utilisateur (Chromium)
const { ipcRenderer } = require('electron');

// Appel vers le main process pour lire un fichier
const content = await ipcRenderer.invoke('read-file', '/path/to/file.txt');
document.getElementById('content').textContent = content;

Avantages et inconvénients

✅ Avantages❌ Inconvénients
Réutilise les compétences webBundle très lourd (Chromium embarqué)
Multiplateforme avec une seule base de codeConsommation mémoire élevée
Accès complet au système de fichiersPas de performances natives pour le rendu
Grande communauté, nombreux exemplesSécurité à gérer manuellement (IPC)
Intégration avec les frameworks modernesStartup time plus long que le natif
