🎯 En bref : Une API REST est une "porte de communication" standardisée entre deux logiciels. Elle permet à une application (le frontend, une app mobile, un autre service) d'envoyer des requêtes à un serveur et d'obtenir des données en retour — comme passer commande dans un restaurant.

Analogie : Imaginez un restaurant. Vous (le client = le frontend) passez commande via le serveur (= l'API). Vous n'avez pas accès à la cuisine (= la base de données) directement. Le serveur transmet votre commande, la cuisine prépare le plat, et le serveur vous rapporte votre repas. REST, c'est le protocole de communication entre vous et le serveur.

Utilisé par : Quasiment toutes les applications web et mobiles modernes. Quand Instagram charge vos photos, quand Spotify envoie votre playlist, quand votre app bancaire affiche votre solde — c'est une API REST (ou similaire) qui travaille en coulisses.

API REST

Une API REST (Representational State Transfer) est une interface standardisée qui permet à deux systèmes informatiques de communiquer via le protocole HTTP — le même protocole que votre navigateur utilise pour charger des pages web.

Les grands principes de REST

1. Ressources et URLs — Tout est une "ressource" identifiée par une URL unique :

  • /users/123 → l'utilisateur n°123
  • /products/456 → le produit n°456
  • /orders → la liste des commandes

2. Les verbes HTTP — L'action à effectuer est exprimée par le type de requête :

VerbeActionExemple
GETLireGET /users/123 → récupérer l'utilisateur 123
POSTCréerPOST /users → créer un nouvel utilisateur
PUT / PATCHModifierPUT /users/123 → mettre à jour l'utilisateur 123
DELETESupprimerDELETE /users/123 → supprimer l'utilisateur 123

3. Sans état (Stateless) — Chaque requête est indépendante. Le serveur ne "se souvient" pas de la requête précédente. Toutes les informations nécessaires (authentification, etc.) doivent être incluses dans chaque requête.

4. Format JSON — Les données échangées sont généralement au format JSON, un format texte lisible :

{
  "id": 123,
  "nom": "Alice Dupont",
  "email": "alice@exemple.fr"
}

Exemple concret

Quand vous ouvrez Twitter/X sur votre téléphone :

  1. L'app envoie GET /timeline?user=alice → demande le fil d'actualité d'Alice
  1. Le serveur vérifie l'authentification, interroge la base de données
  1. Le serveur renvoie un JSON avec les tweets
  1. L'app affiche les tweets à l'écran

Outils pour travailler avec les APIs REST

  • Postman : Application pour tester des APIs — envoyer des requêtes et voir les réponses, sans avoir à coder
  • Insomnia : Alternative à Postman, plus légère
  • Swagger / OpenAPI : Outil pour documenter une API REST de façon standardisée
  • curl : Outil en ligne de commande pour envoyer des requêtes HTTP
💡 REST est le standard le plus répandu pour les APIs web. Comprendre son fonctionnement est indispensable pour travailler avec des développeurs ou intégrer des services externes (paiement, email, cartes, etc.)
