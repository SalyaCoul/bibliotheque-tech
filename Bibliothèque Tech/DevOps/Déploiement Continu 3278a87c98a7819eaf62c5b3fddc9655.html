Déploiement Continu

TypeCI/CD
État
Terminé

🎯 En bref : Le Déploiement Continu (CD — Continuous Deployment/Delivery) est la pratique qui consiste à livrer automatiquement chaque modification de code validée vers un environnement de staging ou de production, sans intervention manuelle.

Analogie : Le déploiement continu est comme une chaîne de production d'usine entièrement automatisée : dès qu'une pièce passe tous les contrôles qualité, elle est automatiquement envoyée en magasin sans que personne ait besoin de signer une autorisation.

Utilisé par : Amazon (déploiements toutes les quelques secondes), Netflix, Facebook, toute équipe cherchant à réduire le time-to-market et le risque lié aux "gros déploiements".

Continuous Delivery vs Continuous Deployment

Continuous DeliveryContinuous Deployment
DéfinitionLe code est toujours prêt à être déployéLe code est automatiquement déployé en production
DéclencheurDéploiement manuel (validation humaine)Déploiement automatique après les tests
Convient àÉquipes avec contraintes réglementairesÉquipes avec fort niveau de confiance dans les tests

Pratiques clés

  • Feature flags : activer/désactiver des fonctionnalités en production sans redéployer
  • Canary deployment : déploiement progressif sur un sous-ensemble d'utilisateurs
  • Blue/Green deployment : deux environnements identiques, bascule instantanée
  • Rollback automatique : retour arrière si les métriques post-déploiement se dégradent
  • GitOps : l'état de la production est déclaré dans Git (Argo CD, Flux)

Outils courants

Jenkins, GitLab CI, GitHub Actions, CircleCI, ArgoCD, Spinnaker, Flux
