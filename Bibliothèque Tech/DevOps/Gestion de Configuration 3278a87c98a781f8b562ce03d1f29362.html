Gestion de Configuration

TypeGestion de Configuration
État
Terminé

🎯 En bref

La gestion de configuration (Configuration Management) est la pratique qui consiste à décrire, versionner et automatiser l'état désiré des serveurs et infrastructures. Au lieu de configurer manuellement chaque machine, on définit ce qu'elle doit être (logiciels installés, fichiers présents, services actifs) et des outils l'y amènent automatiquement.

Analogie

Configurer des serveurs à la main, c'est comme préparer chaque plat d'un restaurant à la main avec chaque cuisinier qui a sa propre recette. La gestion de configuration, c'est le livre de recettes standardisé que tous les cuisiniers suivent à la lettre — chaque plat est identique, reproductible, et si un nouveau cuisinier arrive, il peut recréer n'importe quel plat instantanément.

Utilisé par

Google, Facebook, Twitter, Netflix — toute organisation gérant plus de quelques dizaines de serveurs. Indispensable à partir d'une certaine échelle pour éviter la "configuration drift" (chaque serveur qui diverge subtilement des autres).

Le problème que ça résout : Configuration Drift

Situation SANS gestion de configuration :
Serveur A : nginx 1.18, PHP 8.1, config modifiée manuellement il y a 6 mois
Serveur B : nginx 1.20, PHP 8.2, config "standard"
Serveur C : nginx 1.18, PHP 8.1, patch de sécurité manquant

→ Comportements imprévisibles, bugs impossibles à reproduire

Situation AVEC gestion de configuration :
Tous les serveurs → même état défini dans le code → identiques et prévisibles

Approches : Déclarative vs Impérative

ApprochePrincipeOutilExemple
DéclarativeOn décrit l'état final désiré, l'outil trouve comment y arriverPuppet, Chef, Ansible (en partie)"nginx doit être installé et démarré"
ImpérativeOn décrit les étapes à exécuter dans l'ordreScripts bash, Ansible (tasks)"apt install nginx && systemctl start nginx"

Exemple Ansible (outil le plus populaire)

# playbook.yml — Décrire l'état désiré d'un serveur web
---
- name: Configurer le serveur web
  hosts: web_servers
  become: yes  # sudo

  vars:
    nginx_port: 80
    app_user: deploy

  tasks:
    - name: Installer Nginx
      apt:
        name: nginx
        state: present   # "présent" = installé

    - name: Copier la configuration
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf
      notify: Redémarrer Nginx

    - name: S'assurer que Nginx est démarré
      service:
        name: nginx
        state: started
        enabled: yes     # Au démarrage du serveur

  handlers:
    - name: Redémarrer Nginx
      service:
        name: nginx
        state: restarted
# Appliquer sur tous les serveurs de production
ansible-playbook -i inventaire/prod.yml playbook.yml

# Dry-run : voir ce qui changerait sans appliquer
ansible-playbook -i inventaire/prod.yml playbook.yml --check
