Kubernetes

TypeConteneurisation
État
Terminé

🎯 En bref : Kubernetes (K8s) est un orchestrateur de conteneurs qui automatise le déploiement, la mise à l'échelle et la gestion d'applications conteneurisées en production, sur des clusters de machines.

Analogie : Si Docker est un camion de livraison, Kubernetes est le gestionnaire de flotte — il décide combien de camions déployer, où les envoyer, les remplace automatiquement s'ils tombent en panne, et répartit la charge entre eux.

Utilisé par : Google (créateur original), toutes les grandes plateformes cloud (AWS EKS, GCP GKE, Azure AKS), Netflix, Airbnb, Spotify. Standard de facto pour les architectures cloud-native.

Architecture

  • Cluster : ensemble de machines (nodes) gérées par K8s
  • Control Plane : cerveau du cluster (API Server, Scheduler, etcd)
  • Node : machine (physique ou VM) qui exécute les conteneurs
  • Pod : unité minimale de déploiement — un ou plusieurs conteneurs partageant le même réseau
  • Deployment : définit l'état désiré et gère le cycle de vie des Pods
  • Service : expose les Pods via un endpoint réseau stable
  • Ingress : routage HTTP/HTTPS vers les services internes
  • Namespace : partitionnement logique du cluster pour isoler les environnements
  • ConfigMap / Secret : gestion de la configuration et des données sensibles

Fonctionnalités clés

  • Self-healing : redémarre automatiquement les conteneurs défaillants
  • Auto-scaling : ajuste le nombre de Pods selon la charge (HPA / VPA)
  • Rolling updates : déploiements sans interruption de service
  • Load balancing : distribution automatique du trafic entre Pods
  • Storage orchestration : montage dynamique de volumes (PV/PVC)

Commandes kubectl

kubectl get pods -n production          # Lister les pods d'un namespace
kubectl apply -f deployment.yaml        # Appliquer une configuration
kubectl scale deployment app --replicas=5  # Scaler horizontalement
kubectl rollout status deployment/app   # Suivre un déploiement
kubectl logs <pod-name> -f              # Streamer les logs
kubectl describe pod <pod-name>         # Diagnostiquer un pod
kubectl port-forward svc/app 8080:80    # Accès local à un service

Exemple de Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: mon-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mon-app
  template:
    spec:
      containers:
      - name: mon-app
        image: mon-app:1.0
        ports:
        - containerPort: 3000
