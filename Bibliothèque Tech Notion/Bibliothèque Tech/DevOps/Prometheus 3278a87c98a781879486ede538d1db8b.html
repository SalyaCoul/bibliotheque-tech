Prometheus

TypeSurveillance et Logging
État
Terminé

🎯 En bref : Prometheus est un système de monitoring open-source qui collecte et stocke des métriques sous forme de séries temporelles, avec un puissant langage de requête (PromQL) pour analyser les performances et déclencher des alertes.

Analogie : Prometheus est comme un médecin qui surveille en continu les constantes vitales de tes applications (CPU, mémoire, requêtes/seconde). Il enregistre tout dans un dossier médical horodaté et déclenche une alarme si quelque chose sort des valeurs normales.

Utilisé par : SoundCloud (créateur), CNCF, Netflix, DigitalOcean. Toujours couplé à Grafana pour la visualisation des données.

Architecture

  • Prometheus Server : moteur de collecte, stockage et interrogation des métriques
  • Exporters : agents qui exposent les métriques d'un service tiers (node_exporter, postgres_exporter, blackbox_exporter…)
  • Pull model : Prometheus va chercher les métriques (scraping HTTP) — contrairement à un modèle push
  • Alertmanager : gestion des alertes avec routage vers email, Slack, PagerDuty…
  • PromQL : langage de requête pour interroger et agréger les métriques

Types de métriques

  • Counter : valeur qui ne fait qu'augmenter (nombre de requêtes, erreurs)
  • Gauge : valeur qui monte et descend (usage mémoire, connexions actives)
  • Histogram : distribution de valeurs avec buckets (latence des requêtes)
  • Summary : quantiles calculés côté client (p50, p95, p99)

Exemples PromQL

# Taux de requêtes HTTP sur les 5 dernières minutes
rate(http_requests_total[5m])

# Taux d'erreur (codes 5xx)
rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])

# RAM utilisée en GB
node_memory_Active_bytes / 1024^3

# Services actuellement down
up == 0

# Latence p95 des requêtes
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

Intégration Kubernetes

  • Déploiement via kube-prometheus-stack (chart Helm officiel)
  • Service discovery automatique des pods annotés (prometheus.io/scrape: "true")
  • Alertes prédéfinies pour tous les composants K8s (node disk, pod crash loop…)
  • Métriques exposées sur :9090/metrics par convention
