ElasticSearch

TypeSurveillance et Logging
État
Terminé

🎯 En bref : Elasticsearch est un moteur de recherche et d'analyse distribué open-source, capable d'indexer et d'interroger des volumes massifs de données (logs, textes, documents JSON) avec des temps de réponse quasi instantanés.

Analogie : Elasticsearch est comme un bibliothécaire surpuissant qui a lu tous les livres de la bibliothèque et peut retrouver en millisecondes tous les passages mentionnant un mot-clé, quel que soit le volume — même sur des millions de documents.

Utilisé par : Wikipedia, GitHub, Netflix, LinkedIn, Uber. Cœur de la stack ELK (Elasticsearch + Logstash + Kibana) pour la gestion des logs.

Architecture

  • Cluster : ensemble de nœuds Elasticsearch travaillant ensemble
  • Node : instance unique d'Elasticsearch dans le cluster
  • Index : espace de stockage logique regroupant des documents similaires (comme une table de BDD)
  • Document : unité de base en JSON (comme une ligne en BDD)
  • Shard : fragment d'un index, permettant distribution et parallélisation
  • Replica : copie d'un shard pour la haute disponibilité

Fonctionnalités clés

  • Full-text search : recherche dans le contenu des textes (tokenisation, stemming, fuzzy…)
  • Aggregations : analytics en temps réel (sommes, moyennes, histogrammes)
  • Mapping : typage des champs d'un document (keyword, text, date, geo_point…)
  • Index lifecycle : gestion automatique du cycle de vie des index (hot → warm → cold → delete)
  • Kibana : interface de visualisation et d'exploration (souvent couplée)

Requêtes DSL

GET /logs/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "message": "error" } },
        { "range": { "@timestamp": { "gte": "now-1h" } } }
      ]
    }
  },
  "aggs": {
    "par_service": {
      "terms": { "field": "service.keyword" }
    }
  }
}

Cas d'usage

  • Centralisation de logs : ingestion via Logstash ou Beats, exploration via Kibana
  • Moteur de recherche : recherche produit (e-commerce), recherche full-text
  • Observabilité : APM (Application Performance Monitoring) avec Elastic APM
  • SIEM : détection d'anomalies et analyse de sécurité
