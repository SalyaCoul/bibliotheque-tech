Terminé

🎯 En bref

Proxmox Virtual Environment (Proxmox VE) est une plateforme de virtualisation open source de type 1 (bare-metal) qui combine deux technologies : KVM pour les machines virtuelles complètes et LXC pour les conteneurs légers. Elle s'installe directement sur le matériel et se gère via une interface web intégrée.

Analogie

C'est comme un immeuble de bureaux géré par un syndic ultra-efficace : vous achetez un bâtiment (un serveur physique), et Proxmox vous permet de le diviser en autant de bureaux indépendants (VMs ou conteneurs) que vous voulez, chacun avec ses propres ressources, locataires (OS) et accès réseau — le tout géré depuis un seul tableau de bord.

Utilisé par

Homelabs, PME, hébergeurs, équipes DevOps qui veulent une infrastructure de virtualisation puissante sans les coûts de VMware. Alternative open source sérieuse à VMware ESXi.

Exemple : créer une VM via l'API Proxmox

# Créer une VM (ID 100) via l'API REST Proxmox
curl -X POST "https://proxmox:8006/api2/json/nodes/pve/qemu" \
  -H "Authorization: PVEAPIToken=user@pam!mytoken=xxx" \
  -d "vmid=100" \
  -d "name=mon-serveur-web" \
  -d "memory=2048" \
  -d "cores=2" \
  -d "net0=virtio,bridge=vmbr0" \
  -d "ide2=local:iso/ubuntu-22.04.iso,media=cdrom"

# Démarrer la VM
curl -X POST "https://proxmox:8006/api2/json/nodes/pve/qemu/100/status/start" \
  -H "Authorization: PVEAPIToken=user@pam!mytoken=xxx"

# Lister toutes les VMs et conteneurs du nœud
curl "https://proxmox:8006/api2/json/nodes/pve/qemu" \
  -H "Authorization: PVEAPIToken=user@pam!mytoken=xxx"
