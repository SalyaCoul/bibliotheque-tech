🎯 En bref : npm, pnpm et Yarn sont des gestionnaires de paquets pour JavaScript — l'équivalent de Composer pour PHP. Ils permettent d'installer des milliers de librairies en une commande et de gérer leurs versions.

Analogie : C'est l'App Store de vos projets JavaScript. Au lieu de télécharger manuellement chaque librairie, vous tapez une commande et le gestionnaire se charge de tout : télécharger la bonne version, gérer les dépendances des dépendances, et mettre à jour quand nécessaire.

Utilisé par : Tous les projets JavaScript sans exception — React, Vue, Angular, Node.js, Next.js... Le registre npm (npmjs.com) héberge plus de 2 millions de paquets.

npm / pnpm / Yarn

Ces trois outils font le même travail (gérer les dépendances JavaScript) mais avec des approches différentes.

npm — l'outil officiel

Installé automatiquement avec Node.js. Le plus répandu.

npm install react              # Ajouter une dépendance
npm install --save-dev jest    # Dépendance de dev uniquement
npm install                    # Installer toutes les dépendances
npm run build                  # Lancer un script
npm run dev

Les fichiers clés

package.json — Le "passeport" du projet :

{
  "name": "mon-projet",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "test": "jest"
  },
  "dependencies": {
    "react": "^18.0.0"
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}

package-lock.json / yarn.lock / pnpm-lock.yaml — Verrouillage des versions exactes. À commiter dans Git.

Yarn — l'alternative Facebook

Créé par Meta en 2016, plus rapide que npm à l'époque. Encore très populaire.

yarn add react
yarn add --dev jest
yarn install
yarn build

pnpm — le plus performant

Utilise un store global : les paquets sont partagés entre tous vos projets, ce qui économise l'espace disque et accélère les installations.

pnpm add react
pnpm install
pnpm run dev
