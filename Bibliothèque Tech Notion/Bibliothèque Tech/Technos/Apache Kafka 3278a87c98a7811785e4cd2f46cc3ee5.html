Apache Kafka

Statut
Terminé
TypeMessage Broker

🎯 En bref : Apache Kafka est une plateforme de streaming distribué qui permet d'ingérer, stocker et traiter des flux de données massifs en temps réel — avec une rétention configurable des messages et la capacité de les relire.

Analogie : Kafka est comme un journal d'actualités en temps réel : chaque événement (commande passée, clic, paiement…) est publié comme un article. N'importe quel abonné peut lire ce journal à tout moment — même en remontant dans le passé — sans que l'article disparaisse après lecture.

Utilisé par : LinkedIn (créateur), Uber, Airbnb, Netflix, Spotify. Toute architecture event-driven traitant des volumes importants d'événements.

Concepts clés

  • Topic : canal de publication — les messages y sont classés par sujet
  • Partition : division d'un topic pour la scalabilité et le parallélisme
  • Producer : application qui publie des messages dans un topic
  • Consumer : application qui lit des messages depuis un topic
  • Consumer Group : groupe de consumers se répartissant les partitions d'un topic
  • Broker : serveur Kafka qui stocke les messages
  • Offset : position d'un message dans une partition — Kafka conserve l'historique complet
  • Retention : durée/taille de conservation des messages (ex. 7 jours)

Cas d'usage typiques

  • Event streaming : tracer tous les événements utilisateurs en temps réel
  • Event sourcing : reconstruire l'état d'un système depuis son historique d'événements
  • Data pipeline : alimenter des entrepôts de données (BigQuery, Snowflake) en temps réel
  • Log aggregation : centraliser les logs de tous les microservices
