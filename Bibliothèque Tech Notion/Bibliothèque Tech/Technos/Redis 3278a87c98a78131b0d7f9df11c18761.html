Redis

Statut
Terminé
TypeGestion de cache

🎯 En bref : Redis est un store de données en mémoire open-source extrêmement rapide — il sert à la fois de cache, de base de données clé/valeur temps réel, de gestionnaire de sessions et de broker pub/sub léger.

Analogie : Redis est comme un tableau blanc effaçable dans une salle de réunion : ultra-rapide à lire et écrire, accessible par tout le monde en même temps. Les données ne durent que tant qu'on en a besoin — sauf si on choisit de les persister.

Utilisé par : Twitter, GitHub, Stack Overflow, Airbnb, Instagram. Quasi-universel dans les architectures web modernes pour le cache et les sessions.

Structures de données

Contrairement à Memcached (simple clé/valeur), Redis supporte des structures riches :

  • String : valeur simple (SET user:1:name "Alice")
  • Hash : objet clé/valeur (HSET user:1 name "Alice" age 30)
  • List : liste ordonnée, pile ou file (LPUSH, RPUSH, LPOP)
  • Set : ensemble unique de valeurs (SADD, SISMEMBER)
  • Sorted Set : ensemble avec score pour classer (ZADD, ZRANK) → idéal pour les classements/leaderboards
  • Stream : journal d'événements append-only

Commandes essentielles

SET user:123:session "token_abc"        # Créer une entrée
GET user:123:session                    # Lire
SETEX user:123:session 3600 "token"    # Avec expiration (1 heure)
DEL user:123:session                    # Supprimer
INCR page:home:views                    # Incrémenter atomiquement
ZADD leaderboard 1500 "alice"           # Ajouter au classement avec score
ZRANGE leaderboard 0 9 REV WITHSCORES  # Top 10

Cas d'usage principaux

  • Cache : stocker les résultats de requêtes BDD avec TTL configurable
  • Sessions : conserver les sessions utilisateurs partagées entre plusieurs serveurs
  • Rate limiting : compter les requêtes par IP/utilisateur avec INCR et EXPIRE
  • Pub/Sub : messagerie légère en temps réel entre services
  • Leaderboards : classements triés via les Sorted Sets
