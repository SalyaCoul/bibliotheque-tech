JEST

Statut
Terminé
TypeTesting

🎯 En bref : Jest est le framework de tests JavaScript tout-en-un développé par Meta — il inclut nativement un runner, des assertions, des mocks et la couverture de code, sans configuration préalable.

Analogie : Si Mocha est une cuisine équipée où tu dois apporter tes propres ustensiles, Jest est un restaurant clé en main : tu arrives, tout est là — casseroles, couteaux, recettes, et un chef qui gère tout. Idéal pour démarrer vite sans configuration.

Utilisé par : Facebook/Meta (créateur), projets React, Vue.js, Node.js — standard de facto pour les tests JavaScript front-end et back-end.

Fonctionnalités clés

  • Zero-config : fonctionne immédiatement sur tout projet JS/TS sans setup
  • Assertions intégrées : expect().toBe(), expect().toEqual(), expect().toThrow()…
  • Mocking natif : jest.fn(), jest.spyOn(), jest.mock() — simuler des modules entiers
  • Snapshot testing : capturer le rendu d'un composant React et détecter les régressions visuelles
  • Coverage intégré : rapport de couverture de code avec --coverage (Istanbul sous le capot)
  • Watch mode : relance automatiquement les tests modifiés à chaque sauvegarde

Exemples

// Test unitaire simple
describe('calculerRemise', () => {
  it('applique 10% de remise', () => {
    expect(calculerRemise(100, 10)).toBe(90);
  });

  it('ne descend pas en dessous de 0', () => {
    expect(calculerRemise(10, 200)).toBe(0);
  });
});

// Mock d'un module
jest.mock('./emailService');

test('envoie un email à la création', async () => {
  await creerUtilisateur({ email: 'test@test.com' });
  expect(emailService.sendWelcome).toHaveBeenCalledWith('test@test.com');
});

// Snapshot React
test('rendu du bouton', () => {
  const tree = renderer.create(<Button label="Valider" />).toJSON();
  expect(tree).toMatchSnapshot();
});
