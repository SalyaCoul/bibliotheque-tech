🎯 En bref : GraphQL est une alternative à REST pour construire des APIs. Au lieu de recevoir un paquet de données fixe, le client demande exactement ce dont il a besoin — ni plus, ni moins.

Analogie : Avec une API REST classique, c'est comme commander un menu fixe au restaurant : vous prenez ce qu'on vous sert, même si vous ne voulez pas les frites. Avec GraphQL, c'est comme commander à la carte : "Je veux juste le nom et la photo du profil, pas l'adresse ni le téléphone". Vous récupérez exactement ce que vous avez demandé.

Utilisé par : Facebook (qui l'a créé en 2012), GitHub, Shopify, Twitter, Netflix... GraphQL est particulièrement populaire dans les applications avec des interfaces riches et des besoins de données variés.

GraphQL

GraphQL (Graph Query Language) est à la fois un langage de requête pour les APIs et un environnement d'exécution pour répondre à ces requêtes. Créé par Facebook en 2012, open-sourcé en 2015.

Le problème que GraphQL résout

Avec une API REST classique, on a souvent deux problèmes :

  • Over-fetching : Le serveur renvoie trop de données. Ex : pour afficher juste le nom d'un utilisateur, l'API REST renvoie aussi l'adresse, le téléphone, les préférences... Tout un paquet dont on n'a pas besoin.
  • Under-fetching : Il faut plusieurs requêtes pour avoir toutes les données nécessaires. Ex : pour afficher un profil avec ses articles, il faut d'abord /users/123, puis /users/123/posts, puis /posts/456/comments...

GraphQL règle les deux avec une seule requête flexible.

Comment ça marche

GraphQL utilise un seul endpoint (généralement /graphql) et vous envoyez une requête décrivant exactement ce que vous voulez :

# Requête GraphQL : je veux le nom et l'email de l'utilisateur 123,
# ainsi que les titres de ses 3 derniers articles
query {
  user(id: 123) {
    nom
    email
    articles(limit: 3) {
      titre
      datePublication
    }
  }
}

Le serveur renvoie exactement ça — rien de plus.

REST vs GraphQL — quand utiliser quoi ?

SituationRecommandation
API simple, peu de types de clientsREST ✅
App mobile avec bande passante limitéeGraphQL ✅
Plusieurs clients avec besoins différents (web, mobile, TV...)GraphQL ✅
API publique exposée à des tiersREST (plus universel) ✅
Données avec beaucoup de relations (social, e-commerce...)GraphQL ✅

Outils courants

  • Apollo : La librairie la plus populaire pour intégrer GraphQL côté client (React, Vue...) et serveur (Node.js)
  • GraphiQL / Apollo Studio : Interface visuelle pour explorer et tester une API GraphQL
  • Hasura : Génère automatiquement une API GraphQL depuis une base de données PostgreSQL
💡 GraphQL n'est pas un remplacement universel de REST — c'est un outil complémentaire, particulièrement puissant quand plusieurs types de clients consomment les mêmes données avec des besoins différents.
