🎯 En bref

Mithril est un framework JavaScript minimaliste (~10KB) pour construire des Single Page Applications. Il intègre dans un seul package tout ce dont on a besoin : Virtual DOM, routing et requêtes XHR — sans dépendances externes. Taillé pour la performance et la simplicité.

Analogie

React c'est un restaurant avec une cuisine ouverte où vous assemblez vous-même votre plat en choisissant parmi des dizaines d'ingrédients (bibliothèques). Mithril c'est un menu complet, déjà prêt, avec entrée + plat + dessert — moins de choix, mais tout est là et tout fonctionne ensemble.

Utilisé par

Flems.io, certaines applications académiques et projets à contraintes de taille — très apprécié dans les communautés qui valorisent la minimalisme et les performances.

Syntaxe de base — Composant Mithril

// Pas de JSX requis — hyperscript natif
const Counter = {
    count: 0,

    view() {
        return m('div', [
            m('h2', `Compteur : ${Counter.count}`),
            m('button', {
                onclick: () => Counter.count++
            }, 'Incrémenter'),
            m('button', {
                onclick: () => Counter.count--,
                disabled: Counter.count === 0
            }, 'Décrémenter')
        ]);
    }
};

m.mount(document.body, Counter);

Routing et requêtes HTTP intégrés

// Routing natif
m.route(document.body, '/home', {
    '/home':    HomeComponent,
    '/users':   UsersComponent,
    '/users/:id': UserDetailComponent
});

// Requêtes HTTP natives (équivalent fetch avec gestion auto de l'état)
const Users = {
    list: [],

    oninit() {
        m.request({ method: 'GET', url: '/api/users' })
            .then(data => { Users.list = data; });
    },

    view() {
        return m('ul', Users.list.map(u =>
            m('li', { key: u.id }, u.name)
        ));
    }
};
