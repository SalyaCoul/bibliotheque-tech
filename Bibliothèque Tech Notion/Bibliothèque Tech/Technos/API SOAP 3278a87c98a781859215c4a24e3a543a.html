🎯 En bref

SOAP (Simple Object Access Protocol) est un protocole de communication basé sur XML pour l'échange de données entre applications. Très structuré et verbeux, il impose un contrat strict (WSDL) et supporte nativement les transactions, la sécurité WS-Security et la fiabilité des messages. Standard dans l'entreprise avant l'avènement de REST.

Analogie

Envoyer une requête SOAP c'est comme envoyer un courrier recommandé avec accusé de réception, formulaire officiel en triple exemplaire, et bordereau notarié. REST c'est envoyer un SMS. Les deux transmettent un message, mais SOAP garantit un formalisme et une traçabilité que REST ne fournit pas nativement.

Utilisé par

Systèmes bancaires, assurances, administrations publiques, SAP, Salesforce (anciens services), services de paiement legacy. Encore très présent dans les grandes entreprises et l'intégration B2B.

Structure d'un message SOAP

<!-- Requête SOAP : enveloppe + en-tête + corps -->
POST /service HTTP/1.1
Content-Type: text/xml; charset=utf-8
SOAPAction: "getUserById"

<soap:Envelope
    xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/"
    xmlns:tns="http://example.com/users">

    <soap:Header>
        <tns:AuthToken>Bearer abc123</tns:AuthToken>
    </soap:Header>

    <soap:Body>
        <tns:GetUserRequest>
            <tns:userId>42</tns:userId>
        </tns:GetUserRequest>
    </soap:Body>
</soap:Envelope>
<!-- Réponse SOAP -->
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
    <soap:Body>
        <GetUserResponse>
            <User>
                <Id>42</Id>
                <Name>Alice Martin</Name>
                <Email>alice@example.com</Email>
            </User>
        </GetUserResponse>
    </soap:Body>
</soap:Envelope>

WSDL — Le contrat du service

Le WSDL (Web Services Description Language) décrit le service SOAP : quelles opérations sont disponibles, quels paramètres accepter, quels types de données utiliser. C'est l'équivalent d'un OpenAPI/Swagger pour SOAP.
